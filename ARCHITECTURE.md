## Key Architectural Decisions

### 1. Multi-Tenancy
- **Approach**: Single database, clinic_id filtering
- **Benefit**: Easy deployment, shared infrastructure
- **Data Isolation**: All queries filtered by clinic_id

### 2. Timeline-Centric Design
- **Core Table**: timeline_events
- **Everything flows through timeline**: Events â†’ Templates â†’ Communication
- **Audit Trail**: Complete history with staff attribution

### 3. Acronym Expansion
- **Two-Way Storage**: Original + Expanded
- **Edit Capability**: Can edit original shorthand
- **Display Flexibility**: Show full records to patients

### 4. Role-Based Access
- **JWT Claims**: Include role in token
- **Middleware Check**: requireRole() on routes
- **Frontend Hiding**: Role-based UI rendering

### 5. Stateless API
- **JWT Authentication**: No server-side sessions
- **Horizontal Scaling**: Add more backend containers
- **Cloud-Ready**: Works on any platform

## Performance Optimizations

1. **Database Indexes**: On frequently queried columns
2. **Prisma Client**: Connection pooling built-in
3. **React Optimization**: Zustand for minimal re-renders
4. **Docker Layers**: Cached dependencies
5. **Nginx Compression**: Gzip enabled

## Security Layers

1. **JWT Authentication**: Stateless, tamper-proof
2. **Password Hashing**: bcrypt with salt rounds
3. **Input Validation**: Joi schemas on all inputs
4. **SQL Injection**: Prisma ORM prevents
5. **CORS**: Configured origins only
6. **Role Checks**: Middleware on protected routes

---

*This architecture supports scaling from 1 clinic to 100+ clinics without major changes.*
